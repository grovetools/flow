# Changelog Documentation for Grove Flow

You are documenting Grove Flow's release history and changes.

## Task
Generate a changelog that covers the major releases and updates to Grove Flow.

## Structure
- Use standard changelog format with version numbers
- Include release dates
- Categorize changes as:
  - **Added**: New features
  - **Changed**: Changes in existing functionality  
  - **Deprecated**: Features that will be removed
  - **Removed**: Features that were removed
  - **Fixed**: Bug fixes
  - **Security**: Security improvements

## Content Guidelines
- Start with the most recent releases
- Include significant features like job orchestration, chat integration, plan management
- Mention major architectural changes or API updates
- Keep entries concise but informative
- Focus on user-facing changes rather than internal implementation details

## Format
Use standard semantic versioning (e.g., v1.0.0, v0.9.0) and maintain reverse chronological order.
